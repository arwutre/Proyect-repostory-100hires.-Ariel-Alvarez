# Portfolio Project — Initial Setup

## Tools Installed

- **Cursor IDE** — downloaded from cursor.com. It's essentially VS Code but with native AI integration. I already had some experience with VS Code so the transition was pretty smooth.
- **Claude Code Extension** — installed from Cursor's extensions panel by searching "Claude Code". I had to create an Anthropic account first to log in.
- **Codex Extension** — also installed from the extensions panel by searching "Codex". Once I found the right one, access was straightforward.

## Steps Completed

- Installed Cursor IDE and set it up as my main editor for this project.
- Added and logged into the Claude Code extension.
- Installed and configured the Codex extension.
- Created this public GitHub repository.
- Opened the repository locally in Cursor using `File → Open Folder`.
- Created this `README.md` file documenting the whole process.
- Committed and pushed the changes to the main branch.

## Problems I Ran Into and How I Fixed Them

### 1. Confusion with the Codex extension
When I searched "Codex" in the extensions panel, several options with similar names showed up. I wasn't 100% sure which one was the right one. I solved it by checking the publisher name carefully and reading each extension's description before installing.

### 2. Authentication with Claude Code
When I tried to log in, it asked for credentials I didn't have ready. I had to sign up on anthropic.com first and then go back to the extension to finish the login. Nothing major, just something I didn't anticipate.

### 3. Git not recognized in Cursor at first
The first time I tried to commit from Cursor, it said it couldn't find Git. I had Git installed but Cursor apparently didn't detect it automatically. I fixed it by restarting the editor after confirming Git was properly set in the system PATH.

## Final Thoughts

Overall the process was pretty straightforward. What took me the most time wasn't the installation itself but understanding how all the tools relate to each other and work together. Happy to answer any questions.
